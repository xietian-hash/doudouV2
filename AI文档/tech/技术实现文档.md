## 4.3 统一响应 Envelope

成功响应：
```json
{ "code": 0, "message": "ok", "data": {}, "traceId": "018f0d8b9d2c4b91" }
```

失败响应：
```json
{ "code": 10001, "message": "VALIDATION_ERROR", "details": {}, "traceId": "018f0d8b9d2c4b91" }
```

分页响应（data 字段）：
```json
{ "pageNo": 1, "pageSize": 20, "total": 123, "list": [] }
```

# 六、接口契约概览

接口前缀：`/api`，认证方式：除微信登录外均使用 `Authorization: Bearer <JWT>`。

## 6.1 接口分组

| 标签 | 说明 |
| -- | -- |
| Auth | 微信登录、获取当前用户 |
| Accounts | 账户增删改查、设默认 |
| Categories | 分类增删改查、分类图标查询 |
| Tags | 标签增删改查 |
| Bills | 账单增删改查、日历汇总、批量新增 |
| Voice | 语音文本解析 |
| Statistics | 分类支出统计 |

## 6.2 关键接口说明

### 微信登录 `POST /api/auth/wechat-login`

- 无需鉴权
- 入参：`{ wxCode, nickname?, avatarUrl? }`
- 出参：`{ token: JWT, user: User }`
- 后端用 wxCode 换取 openid，首次创建用户并初始化种子数据，返回 JWT

### 账单列表 `GET /api/bills`

- 支持参数：`month`（YYYY-MM）、`date`（YYYY-MM-DD）、`pageNo`、`pageSize`
- 返回：分页对象，含分类图标、二级分类名、账户名、金额

### 日历汇总 `GET /api/bills/calendar-summary`

- 参数：`month`（必填，YYYY-MM）
- 返回：每日收支汇总数组 `[{ date, expenseAmount, incomeAmount }]`

### 批量新增账单 `POST /api/bills/batch`

- 语音解析确认后调用
- 事务保存，任一失败整体回滚，同时更新账户余额

### 语音解析 `POST /api/voice/parse`

- 入参：`{ text: string }`（语音识别后的原始文本）
- 出参：结构化账单数组 `VoiceParsedBill[]`，含 `confidence`、`needsConfirm` 字段

### 分类统计 `GET /api/statistics/category-expense`

- 参数：`month`（必填，YYYY-MM）
- 返回：各叶子分类的支出金额和占比

# 七、认证与安全

## 7.1 微信登录流程

```
小程序 → wx.login() → 获取 wxCode
小程序 → POST /api/auth/wechat-login { wxCode }
后端 → 调用微信接口 → 换取 openid
后端 → 查找或创建 users.openid 用户
后端 → 首次创建时初始化种子数据（账户、分类）
后端 → 签发 JWT → 返回 { token, user }
小程序 → wx.setStorageSync('token', token)
后续请求 → Authorization: Bearer <token>
```

## 7.2 数据隔离

- 所有业务表查询和写入必须携带当前 `userId` 条件
- 后端从 JWT 解析用户，不信任前端传入的 `userId`
- 跨用户访问返回 403

## 7.3 敏感信息管理

- JWT secret、微信 app secret、大模型 key、服务器私钥不得提交 Git
- 配置文件按本地/测试/生产环境分离

# 八、关键业务规则

## 8.1 账户余额联动

| 操作 | 余额变更 |
| -- | -- |
| 新增支出账单 | 账户余额 -= 金额 |
| 新增收入账单 | 账户余额 += 金额 |
| 编辑账单 | 账户余额按修改前后差额调整 |
| 删除账单 | 账户余额回滚（反向操作） |
| 批量新增账单 | 所有操作在同一事务内完成 |

> 所有余额变更必须与账单写入在同一事务内完成（Prisma transaction）。

## 8.2 分类约束

- 账单只可挂叶子分类（无子分类节点）
- 分类最多两级
- 有子分类或关联账单时不可删除分类
- 有关联账单时仍可编辑分类名称和图标
- 保存账单时更新叶子分类的 `last_used_at`

## 8.3 账户约束

- 同用户下账户名称唯一
- 有关联账单时不可删除账户
- 同用户下只允许一个默认账户，设置后原默认自动取消

## 8.4 标签约束

- 同用户下标签名称唯一
- 删除标签时同步删除 `bill_tags` 关联记录
- 当前版本单笔账单只支持关联一个标签

## 8.5 语音解析流程

```
1. 小程序录音 → wx.RecorderManager
2. 调用微信语音识别 API 或小程序内置识别 → 获取文本
3. POST /api/voice/parse { text }
4. 后端调用大模型（OpenAI 兼容格式）解析账单结构
5. 匹配用户分类树中的叶子分类
6. 返回 VoiceParsedBill[]（含 confidence、needsConfirm）
7. 单条直接回填记账表单；多条进入确认记账抽屉
8. 用户确认后 POST /api/bills/batch 批量保存
```

## 9.2 分包策略

- 主包 `pages/`：记录页、记账页、我的页，体积控制在 1.5MB 以内
- 低频分包：账户管理、分类管理、标签管理、账单详情，在 `app.json` 统一声明

## 9.3 请求封装规范

`services/request.js` 负责：
- 自动注入 `Authorization: Bearer <token>`
- 自动生成并携带 `traceId`（前端生成 UUID）
- 自动解包 Envelope，取 `data` 字段
- 统一处理 401（跳转登录）、403（提示无权限）、网络错误（Toast 提示）

## 9.4 认证流程

```
小程序启动 → 读取 wx.getStorageSync('token')
token 存在 → 校验 JWT 有效期
token 不存在或失效 → wx.login() → 获取 wxCode → POST /api/auth/wechat-login
登录成功 → 保存 token → 拉取当前用户信息 → 进入主页
```
