## 录音按钮交互修复（追加）

### 缺陷描述

- 用户快速点击底部录音按钮后，录音面板出现且无法消失（UI 卡死）。
- 时长太短的录音也走了上传流程，未给到「按下/松开」的长按反馈。

### 根因

- 录音面板可见性 `recording` 仅在 `recorder.onStop` 回调中复位；`recorder.start()` 启动尚未完成时 `recorder.stop()` 不会触发 `onStop`，导致 `recording=true` 卡住。
- 时长判定依赖 `res.duration`，与 `onStop` 强耦合，回调不触发就无任何反馈。

### 修复方案

- 触控状态机迁移到 `custom-tab-bar/index.js`：触摸开始记录时间戳与起点 Y 坐标；touchmove 计算 `deltaY < -60px` 切换取消态；touchend / touchcancel 计算 `duration`，向 bill 页发出 `voiceRecord:stop {duration, canceled}`。
- `pages/bill/index.js` 不再依赖 `res.duration`；`stopRecording(payload)` 先无条件 `setData({recording:false, recordingCanceling:false})`，根据 `canceled` 与 `duration` 决定是取消、超短提示还是正常上传；用 `_abortVoiceUpload` 标志拦截 `onStop` 中的上传副作用。
- 新增取消态可视化：录音面板 / 波形条 / 「上滑取消」「松开完成」提示均按取消态切换为橙红配色与权重。
- 同步在 `AI文档/UI规范.md` 中追加「录音按钮交互（语音记账）」一节，明确交互模型、阈值与实现要点。

## 解析中浮层（追加）

### 缺陷描述

- 完成一次录音后立即再次按下录音按钮无反应；原因是 `voiceParsing` 守卫将 FAB 锁定，但用户无任何视觉反馈。

### 修复方案

- 录音结束到「确认记账」抽屉出现之间（含语音转文字与文字转 JSON 两个阶段），在 `pages/bill` 渲染全屏浮层提示「兜兜思考中...」。
- 浮层使用 `catchtap` + `catchtouchmove` 拦截下层交互，FAB 因此自然无法触发新录音，无需引入请求中断逻辑。
- 浮层内白卡居中，三个品牌色 `#5DBE88` 圆点错相位呼吸动画，配合主文字色 `#2B4A3A` 文案「兜兜思考中...」。
- 解析成功 → 浮层消失 → 弹出「确认记账」抽屉；解析失败 → 浮层消失 → 错误 toast 提示。
- 同步在 `AI文档/UI规范.md` 「录音按钮交互」节追加「解析中浮层」子节。
