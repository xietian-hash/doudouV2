> 本文档用于拆解开发任务，任务来源为 `AI文档/PRD.md`、`AI文档/UI规范.md`、`AI文档/开发规范.md`。任务执行前应先确认接口契约、数据库结构和 UI 规范是否已同步更新。

# 一、任务总览

## 目标

完成“兜兜有钱 v2”微信记账小程序一期能力：微信登录、手工记账、语音记账、账单记录、账户管理、分类管理、标签管理和基础统计。

## 技术范围

| 范围 | 技术选型 | 说明 |
| -- | -- | -- |
| 小程序端 | Taro + React | 负责页面、交互、状态管理、接口调用 |
| 后端服务 | Node.js + NestJS | 负责鉴权、业务规则、数据权限、语音解析对接 |
| 数据库 | MySQL + Prisma | 负责数据模型、迁移、种子数据 |
| 接口契约 | OpenAPI 3.0 | 前后端联调的单一事实来源 |
| 部署 | Docker + Docker Compose + GitHub Actions + Nginx | 一期先完成基础部署链路 |

## 优先级

| 优先级 | 定义 |
| -- | -- |
| P0 | 一期上线必须完成 |
| P1 | 一期可用体验需要完成，允许按排期拆分 |
| P2 | 可延后优化或二期完善 |

# 六、建议开发顺序

1. 完成工程初始化、OpenAPI、数据库模型和种子数据。
2. 完成后端登录、账户、分类、标签、账单基础接口。
3. 完成小程序全局主题、底部导航、记录页和手工记账页。
4. 完成账户/标签/日期/分类抽屉和新增/编辑弹窗。
5. 完成语音记账解析、确认记账抽屉和批量保存。
6. 完成我的页、管理页和分类统计。
7. 做全量联调、自测、UI 规范走查和安全检查。

# 七、任务状态维护规则

1. 任务状态使用：`待开始`、`进行中`、`已完成`、`阻塞`、`延期`。
2. 每次完成代码任务后，同步更新对应文档：PRD、OpenAPI、UI 规范或任务文档。
3. 若任务实现与 PRD 或 UI 规范不一致，先更新设计/需求并确认，再开发。
4. 任何涉及接口字段、数据表、权限规则的变更，必须同步 OpenAPI 和数据库迁移。
5. 每个 P0 任务完成后必须自测；提交前必须检查敏感文件和编码格式。
