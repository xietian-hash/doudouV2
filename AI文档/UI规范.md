> 本文件只记录 UI 规范。UI 规范用于建立统一、可复用、可落地的界面规则，保障视觉一致性和交互体验连贯性；与 UI 无关的业务逻辑不记录。

# 一、设计原则

1. 用户体验优先于技术偏好、代码整洁度、架构优雅度和代码复杂度。
2. 为目标设计，不为功能设计：先明确用户要完成什么，再决定界面如何承载。
3. 不让用户思考：核心操作应直接可见、状态明确、反馈及时。
4. 渐进式展示：先展示高频核心信息，低频字段通过抽屉、弹窗或管理页承载。
5. 系统承担复杂性：能自动默认的不让用户重复选择，能即时选择的不二次确认。
6. 所有弹窗、抽屉、按钮、列表、导航必须遵循本规范，不允许页面各自发明样式。

# 二、视觉风格

## 设计定位

产品采用“清新治愈风”，整体偏轻量、干净、柔和。界面以浅绿色背景、白色内容面、低对比描边和克制阴影为主，适合日常高频记账场景。

## 录音按钮交互（语音记账）

### 交互模型

录音按钮采用「按下即录、松开即停」的长按交互（**非点击切换**）：

1. **按下**（touchstart）：立即显示录音面板并开始录音，无任何感知延迟。
2. **按住**（touchmove）：持续录音；若向上滑动超过 `60px`，进入「取消态」（面板与提示变红）。
3. **松开**（touchend）：
   - 已进入取消态 → 销毁录音、提示 `已取消`，不上传。
   - 按住时长 `< 1000ms` → 销毁录音、提示 `录音时间太短，请长按说话`，不上传。
   - 按住时长 `≥ 1000ms` 且未取消 → 停止录音并上传解析。
4. **异常**（touchcancel / recorder.onError）：等同取消，统一兜底关闭录音面板。

### 视觉状态

| 状态 | 背景 | 波形条颜色 | 「上滑取消」提示 | 「松开完成」提示 |
| -- | -- | -- | -- | -- |
| 正常录音 | `#FFFFFF` + `#E8F3EE` 描边 | 品牌绿渐变 `#5DBE88` 系列 | `opacity 0.4` 弱化 | `opacity 1` 高亮 |
| 取消态 | `#FFF5F0` + `#FFB59A` 描边 | 警告橙渐变 `#FF7043` 系列 | `opacity 1` 高亮，文案改为「松开取消」 | `opacity 0.4` 弱化 |

### 实现要点

1. 录音面板可见性（`recording`）由触控事件直接控制，**不依赖** `recorder.onStop` 回调，避免快速点击时 UI 卡死。
2. 时长判定基于 `Date.now()` 的触控时间戳差值（touchstart → touchend），不使用 `res.duration`，避免 recorder 启动延迟干扰。
3. 取消/超短场景下，仍需调用 `recorder.stop()` 释放硬件资源，并用 `_abortVoiceUpload` 标志拦截 `onStop` 中的上传副作用。
4. 上滑阈值默认 `60px`，可根据真机测试在 `50-80px` 之间调整。

### 解析中浮层

录音结束到「确认记账」抽屉出现之间，存在「上传 + 语音转文字 + 文字转 JSON」的等待期。该期间必须展示全屏浮层，避免用户误以为按钮失灵：

| 属性 | 值 |
| -- | -- |
| 触发条件 | `voiceParsing === true`（录音上传开始到响应返回之间） |
| 遮罩 | 全屏 `fixed`，背景 `rgba(43, 74, 58, 0.4)`（与抽屉遮罩一致），`z-index: 100` |
| 卡片 | 居中白卡，圆角 `24rpx`，padding `36rpx 48rpx`，柔和投影 `0 12rpx 32rpx rgba(43, 74, 58, 0.18)` |
| 内容 | 三个品牌色 `#5DBE88` 圆点（`14rpx`）错相位呼吸动画 + 文案「兜兜思考中...」 |
| 文字样式 | `Noto Sans SC` `28rpx / 600`，颜色 `#2B4A3A`，字距 `1rpx` |
| 动画 | `1.1s ease-in-out infinite`，三圆点延迟 `0 / 0.18 / 0.36s`，缩放 `0.6 → 1`、透明度 `0.4 → 1` |

#### 行为规则

1. 浮层期间 `catchtap` + `catchtouchmove` 拦截所有下层交互，FAB 因此自然无法触发新录音。
2. 解析成功 → 浮层消失 → 弹出「确认记账」抽屉。
3. 解析失败 → 浮层消失 → 错误 toast 提示，用户可重新长按录音。
4. 浮层不阻断原生 tabBar 切换；若用户切走再切回时仍在解析中，浮层依然展示。

## 列表项

1. 列表卡片背景白色，圆角 `16px`，描边 `#E8F3EE`。
2. 列表项左右内边距 `16px`，上下内边距 `12px`。
3. 左侧图标容器通常为 `40px × 40px`，圆形或 `20px` 圆角浅底。
4. 金额列右对齐；支出金额使用绿色体系，收入金额使用红色体系。

## 分类网格

1. 分类网格按行排列，每个单元高度建议 `62px`。
2. 单元内图标在上、分类名在下，间距 `4px`。
3. 选中态使用 `#EAF7EF` 背景、`#5DBE88` 描边，分类名使用 `#2B8A57`。
4. 未选中态背景透明或浅底，文字使用 `#6F9383/#7A9E8E`。

## 数字键盘

1. 键盘背景白色，顶部圆角 `[8px, 8px, 0, 0]`，描边 `#E8F3EE`。
2. 数字键背景 `#F7FAF8`，圆角 `8px`，高度 `36px`，字号 `16px / 700`。
3. 键间间距 `6px`（横向、纵向一致）。
4. 右侧操作区宽度 `72px`、高度 `162px`：上方删除键（`36px` 高），下方完成键（占满剩余高度）。
5. 删除键使用警告浅底 `#FEF3E8` 和警告色 `#F5A25A`。
6. 完成键使用主色背景，圆角 `12px`，对勾字号 `18px`、文案字号 `10px`，文字和图标白色。
7. 整体键盘高度约为「数字键 4 行 + 行间距」≈ `162px`，按 70% 比例压缩，避免过度占用页面空间，腾出弹性留白给上半部分。

## 记账页备注输入

1. 备注采用独立输入行，**不放入元信息胶囊行**，不弹原生 modal。
2. 输入行结构：左侧铅笔图标（`✎`，色 `#7A9E8E`）+ 右侧 `input` 输入框（占满剩余宽度）。
3. 容器：白底、高度 `44px`、左右内边距 `20px`、无描边。
4. 占位文字：「写点备注...」，颜色 `#B5D0C6`，字号 `13px`。
5. 输入文字：颜色 `#2B4A3A`，字号 `13px`，单行；最长 `100` 字符。
6. 元信息行（账户/标签/日期）改为 `3` 列等宽 grid，不再包含备注胶囊。

## 记账页页面布局（三段式）

1. 记账页采用「**顶部固定 / 中间滚动 / 底部固定**」三段式结构，整页禁止系统级滚动。
2. 顶部固定区（始终可见）：类型 Tab（支出/收入）+ 金额输入面板。
3. 中间滚动区：分类宫格用 `scroll-view` 撑满，可独立上下滑查看全部叶子分类。
4. 底部固定区：备注输入行 + 元信息胶囊行 + 数字键盘（用 `position: fixed`）。
5. 实现要点：`page` 与 `.bill-page` 均设 `height: 100vh`，`page` 加 `overflow: hidden` 关闭系统滚动；`.category-scroll` 用 `flex: 1; min-height: 0;` 撑满中间空间。

## 弹窗

1. 居中弹窗宽度 `335px`，背景白色，圆角 `20px`。
2. 遮罩使用 `#2B4A3A`，透明度 `0.4` 到 `0.45`。
3. 弹窗结构统一为：标题区、内容区、底部按钮区。
4. 标题使用 `18px / 800`，右上角关闭图标按具体场景决定；若底部已有取消按钮，可不展示关闭图标。
5. 底部按钮统一高度 `42px` 到 `44px`，取消在左、确认在右。

## 底部抽屉

1. 抽屉宽度铺满屏幕，背景白色，顶部圆角 `[20px, 20px, 0, 0]`。
2. 抽屉阴影使用柔和上投影，颜色 `#2B4A3A18`，模糊 `20px`，偏移 `0, -6`。
3. 抽屉标题使用 `18px / 800`。
4. 账户、标签、日期选择均使用抽屉，不使用普通弹窗。
5. 账户和标签抽屉底部按钮并排展示：左侧管理/新建，右侧取消。
6. 日期抽屉底部左侧为取消，右侧为今天；点击日期后立即选择并关闭。
7. 抽屉最小高度统一为 `884rpx`（以日期选择抽屉 6 行日历完整展开的高度为准），最大高度 `calc(86vh - safe-area)`，内容不足时底部自然留白，避免抽屉高度因内容多寡而抖动。
8. 按钮区始终贴抽屉底部，内容不足时上方自然留白（实现：`.sheet` 用 flex 纵向布局，`.sheet-footer` 用 `margin-top: auto`）。

### 使用规则

1. 任何列表/网格区域无数据时必须展示空状态组件，不允许出现纯白屏。
2. 主文案描述当前空态的事实，副文案给出下一步操作引导（动词开头）。
3. 列表内嵌空态时整体居中，组件至少距列表顶部 `48px`。
4. 全屏空态时组件垂直方向居中，避开顶部导航和底部 Tab Bar。
5. 不使用阴影、渐变和强对比插画，保持治愈轻量风格。
6. 加载中状态使用 Skeleton 而非空状态组件，加载失败使用错误 Toast + 重试入口。
