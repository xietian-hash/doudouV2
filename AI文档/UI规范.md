> 本文件只记录 UI 规范。UI 规范用于建立统一、可复用、可落地的界面规则，保障视觉一致性和交互体验连贯性；与 UI 无关的业务逻辑不记录。

# 一、设计原则

1. 用户体验优先于技术偏好、代码整洁度、架构优雅度和代码复杂度。
2. 为目标设计，不为功能设计：先明确用户要完成什么，再决定界面如何承载。
3. 不让用户思考：核心操作应直接可见、状态明确、反馈及时。
4. 渐进式展示：先展示高频核心信息，低频字段通过抽屉、弹窗或管理页承载。
5. 系统承担复杂性：能自动默认的不让用户重复选择，能即时选择的不二次确认。
6. 所有弹窗、抽屉、按钮、列表、导航必须遵循本规范，不允许页面各自发明样式。

# 二、视觉风格

## 设计定位

产品采用“清新治愈风”，整体偏轻量、干净、柔和。界面以浅绿色背景、白色内容面、低对比描边和克制阴影为主，适合日常高频记账场景。

1. 页面主背景使用 `#F0FBF5` 或 `#F6FBF8`，内容承载面使用白色。
2. 品牌主色只用于主操作和当前选中态，不大面积铺满页面。
3. 支出在当前 UI 中使用绿色体系表达；收入使用红色体系表达。
4. 描边优先使用 `#E8F3EE`，避免高对比黑灰边框。
5. 遮罩使用 `#2B4A3A`，透明度 `0.36` 到 `0.45`。

# 三、字体规范

## 字体

| 类型 | 字体 |
| -- | -- |
| 中文 | `Noto Sans SC`、`PingFang SC`、`Microsoft YaHei` |
| 数字金额 | `Inter` |

## 字号

| 样式 | 字号/字重 | 使用场景 |
| -- | -- | -- |
| Display | `32px / 800` | 大金额、统计金额 |
| H1 | `22px / 700` | 页面大标题 |
| H2 | `18px / 600-800` | 抽屉标题、卡片标题、重要分类名 |
| H3 | `16px / 600` | 列表主标题、模块标题 |
| Body | `14px / 400` | 正文、普通说明 |
| Button | `14px / 600` | 常规按钮文案 |
| Caption | `12px / 400-600` | 辅助说明、分类网格文字 |
| Micro | `10px / 400` | 底部导航标签、极弱备注 |

## 排版规则

1. 页面标题保持全局一致，不同页面不自行放大或缩小。
2. 金额数字优先使用 `Inter`，金额主数字可使用 `32px` 到 `36px`、字重 `800`。
3. 记账页二级分类名需要显眼，使用 `18px / 800`。
4. 按钮文字不小于 `13px`，主按钮建议 `14px / 600`。
5. 不使用负字距；中文文案保持自然字距。

# 四、间距规范

| 名称 | 数值 | 使用场景 |
| -- | -- | -- |
| XS | `4px` | 图标与文字的极小间距、网格内部间距 |
| S | `8px` | 同组元素间距、键盘按键间距 |
| M | `12px` | 按钮内图标文字间距、卡片内部小间距 |
| L | `16px` | 页面左右边距、抽屉内边距、列表间距 |
| XL | `20px` | 弹窗/抽屉顶部和底部内边距 |
| XXL | `24px` | 大模块间距、弹窗按钮区间距 |

## 布局规则

1. 移动端页面宽度按 `375px` 设计，页面内容左右边距优先使用 `16px`。
2. 抽屉和弹窗内容内边距使用 `16px` 到 `20px`。
3. 列表项高度不低于 `52px`，可点击区域不低于 `42px`。
4. 分类网格项高度建议 `62px`，图标和文字垂直排列。
5. 底部导航本体高度固定 `56px`，凸起录音按钮不改变导航本体高度。

# 五、圆角规范

| 名称 | 数值 | 使用场景 |
| -- | -- | -- |
| XS | `4px` | 小角标、下划线、微型标记 |
| S | `8px` | 数字键盘按键、小型分类格 |
| M | `12px` | 按钮、输入框、账户项、标签项 |
| L | `16px` | 列表卡片、较大信息容器 |
| XL | `20px` | 主卡片、居中弹窗、底部抽屉 |
| Pill | `24px` 及以上 | Toast、胶囊标签、圆形按钮 |

## 使用规则

1. 普通按钮和输入框统一使用 `12px` 圆角。
2. 底部抽屉顶部圆角使用 `[20px, 20px, 0, 0]`。
3. 居中弹窗使用 `20px` 圆角。
4. 底部导航本体在记账页保持无圆角，录音按钮单独凸起悬浮。

# 六、按钮规范

## 按钮类型

| 类型 | 样式 | 使用场景 |
| -- | -- | -- |
| 主要按钮 | 背景 `#5DBE88`，文字白色，圆角 `12px` | 确认、保存、完成 |
| 次要按钮 | 白底或浅底，文字 `#5DBE88/#7A9E8E`，描边 `#E8F3EE` | 取消、返回、辅助操作 |
| 危险按钮 | 背景 `#FF4444`，文字白色 | 删除、不可逆操作确认 |
| 文字按钮 | 无背景，文字品牌色 | 轻量入口、查看更多 |
| FAB | 圆形主色按钮，带柔和阴影 | 中间记账/录音入口 |

## 按钮尺寸

1. 常规按钮高度 `42px` 到 `44px`。
2. 抽屉底部并排按钮高度 `42px`，间距 `10px`。
3. 底部导航中间凸起录音按钮为 `60px × 60px`，图标 `28px`。
4. 图标按钮触达区域不小于 `40px × 40px`。

## 交互规则

1. 主操作放在右侧或底部更靠近拇指的位置；取消操作放左侧或次级位置。
2. 日期选择采用点击即选择，不需要再次确认；底部按钮只保留“取消”和“今天”。
3. 抽屉中如已有管理/新建按钮，取消按钮与其并排展示。

# 七、输入框规范

| 状态 | 样式 |
| -- | -- |
| 默认 | 白底，描边 `#E8F3EE`，圆角 `12px` |
| 聚焦 | 白底，描边 `#5DBE88`，圆角 `12px` |
| 错误 | 白底，描边 `#FF4444`，下方展示错误提示 |

输入框内边距使用垂直 `12px`、水平 `16px`；占位文字使用 `#B5D0C6`。

# 八、图标规范

## 图标来源

1. 分类图标使用 Unicode Emoji，来自 `category_icons` 分类图标库，可被多个分类复用。
2. 账户、导航、操作类图标固定在前端组件中，不进入分类图标库。
3. 操作类图标优先使用 Lucide 风格，线宽统一为 `2px`。

## 图标尺寸

| 场景 | 尺寸 |
| -- | -- |
| 底部导航图标 | `22px` |
| 元信息胶囊图标 | `14px` |
| 抽屉列表图标 | `20px` |
| 弹窗/Toast 图标 | `16px` |
| 录音按钮图标 | `28px` |
| 分类 Emoji | `20px` |

## 使用规则

1. 分类选择、确认记账、账单列表必须展示分类图标。
2. 分类只展示二级分类时，图标与二级分类保持同一信息组。
3. 未选中操作图标使用 `#B5D0C6` 或 `#7A9E8E`；选中图标使用 `#5DBE88` 或 `#2B8A57`。

# 九、组件规范

## 底部导航

1. 导航本体高度固定 `56px`，背景 `#FFFFFF`，顶部描边 `#E8F3EE`。
2. 非记账页中间按钮默认显示加号；记账页显示麦克风。
3. 记账页录音按钮采用凸起悬浮样式：按钮 `60px × 60px`，圆形，背景 `#5DBE88`，白色麦克风图标，阴影 `#5DBE8870`。
4. 录音按钮凸起不改变导航本体高度，导航外层可使用 `68px` 承载按钮凸出区域。
5. 未选中导航文字和图标使用 `#B5D0C6`，选中态使用 `#5DBE88`。

## 列表项

1. 列表卡片背景白色，圆角 `16px`，描边 `#E8F3EE`。
2. 列表项左右内边距 `16px`，上下内边距 `12px`。
3. 左侧图标容器通常为 `40px × 40px`，圆形或 `20px` 圆角浅底。
4. 金额列右对齐；支出金额使用绿色体系，收入金额使用红色体系。

## 分类网格

1. 分类网格按行排列，每个单元高度建议 `62px`。
2. 单元内图标在上、分类名在下，间距 `4px`。
3. 选中态使用 `#EAF7EF` 背景、`#5DBE88` 描边，分类名使用 `#2B8A57`。
4. 未选中态背景透明或浅底，文字使用 `#6F9383/#7A9E8E`。

## 数字键盘

1. 键盘背景白色，顶部圆角 `[8px, 8px, 0, 0]`，描边 `#E8F3EE`。
2. 数字键背景 `#F7FAF8`，圆角 `8px`，高度 `52px`。
3. 删除键使用警告浅底 `#FEF3E8` 和警告色 `#F5A25A`。
4. 完成键使用主色背景，圆角 `12px`，文字和图标白色。

## 弹窗

1. 居中弹窗宽度 `335px`，背景白色，圆角 `20px`。
2. 遮罩使用 `#2B4A3A`，透明度 `0.4` 到 `0.45`。
3. 弹窗结构统一为：标题区、内容区、底部按钮区。
4. 标题使用 `18px / 800`，右上角关闭图标按具体场景决定；若底部已有取消按钮，可不展示关闭图标。
5. 底部按钮统一高度 `42px` 到 `44px`，取消在左、确认在右。

## 底部抽屉

1. 抽屉宽度铺满屏幕，背景白色，顶部圆角 `[20px, 20px, 0, 0]`。
2. 抽屉阴影使用柔和上投影，颜色 `#2B4A3A18`，模糊 `20px`，偏移 `0, -6`。
3. 抽屉标题使用 `18px / 800`。
4. 账户、标签、日期选择均使用抽屉，不使用普通弹窗。
5. 账户和标签抽屉底部按钮并排展示：左侧管理/新建，右侧取消。
6. 日期抽屉底部左侧为取消，右侧为今天；点击日期后立即选择并关闭。

## Toast 全局提示

### 使用场景

| 类型 | 触发时机 | 示例文案 |
| -- | -- | -- |
| 成功 | 添加、编辑、删除操作完成 | 添加成功、编辑成功、删除成功 |
| 失败 | 操作异常、网络错误 | 操作失败，请重试 |

### 视觉规范

| 属性 | 值 |
| -- | -- |
| 背景色 | 成功 `#1AAD19`，失败 `#FF4444` |
| 圆角 | `24px` |
| 内边距 | 垂直 `8px`，水平 `16px` |
| 图标 | `16px × 16px`，白色；成功用 `circle-check`，失败用 `circle-x` |
| 文字 | `14px`，`#FFFFFF`，Regular |
| 布局 | 图标 + 文字水平排列，gap `6px`，垂直居中 |
| 最大宽度 | `240px` |
| 位置 | 屏幕垂直居中，水平居中 |

### 交互规范

1. 显示时长：`2000ms` 后自动关闭。
2. 动画：淡入 `200ms`，淡出 `200ms`。
3. 同一时刻只显示一个 Toast，新 Toast 替换旧 Toast。
4. 显示期间不阻断页面交互。
