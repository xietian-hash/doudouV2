> 本文件只记录UI规范，UI规范是为产品界面建立的一套统一、可复用、可落地的设计规则集合，核心目的是保障界面视觉一致性、交互体验连贯性，与UI规范无关的内容不记录

## 交互设计原则

1. 用户体验是所有产品的最高准则，优先级高于技术偏好、代码整洁度、架构优雅度、代码复杂度
2. 为目标设计，不为功能设计：先问「用户要完成什么」，再决定怎么实现
3. 不要让用户思考：交互应该不言自明。需要说明书才能用，设计就是失败的
4. 系统承担复杂性：能自动化的不手动，能推断的不让用户填，能一步完成的不要拆成三步
5. 渐进式展示：先给核心，细节按需展开。不要一次性把所有选项甩给用户
6. 反馈引导行动：不要只报告问题("连接失败")，要引导下一步("正在重试，预计5 秒后恢复")

### 图标风格

*   线性图标：描边宽度统一为 2px，风格简洁，用于常规操作
*   面性图标：填充色为品牌主色 / 中性色，用于重要功能、选中状态
*   色图标：主色 + 中性色组合，用于特殊场景强调

## Toast 全局提示

### 使用场景

| 类型 | 触发时机 | 示例文案 |
| -- | -- | -- |
| 成功 | 添加、编辑、删除操作完成 | 添加成功、编辑成功、删除成功 |
| 失败 | 操作异常、网络错误 | 操作失败，请重试 |

### 视觉规范

| 属性 | 值 |
| -- | -- |
| 背景色 | 成功 `#1AAD19`，失败 `#FF4444` |
| 圆角 | 24px（pill 胶囊形） |
| 内边距 | 垂直 8px，水平 16px |
| 图标 | 16×16，白色；成功用 `circle-check`，失败用 `circle-x` |
| 文字 | 14px，`#FFFFFF`，Regular |
| 布局 | 图标 + 文字水平排列，gap 6px，垂直居中 |
| 最大宽度 | 240px |
| 位置 | 屏幕垂直居中，水平居中 |

### 交互规范

*   显示时长：2000ms 后自动关闭
*   动画：淡入 200ms / 淡出 200ms
*   同一时刻只显示一个 Toast，新 Toast 会替换旧 Toast
*   显示期间不阻断页面交互
